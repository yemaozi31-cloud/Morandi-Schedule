## ☁️ 坚果云同步配置

> 所有数据存储在你自己电脑/手机本地，通过坚果云在多设备之间同步。
> **数据不上传我们的服务器，只有你能看到。**

### 第一步：注册坚果云

如果你还没有坚果云账号：

1. 打开 [坚果云官网](https://www.jianguoyun.com)
2. 点击右上角「注册」→ 用邮箱注册
3. 免费版每月 1GB 上传流量 + 3GB 下载流量，**个人用完全足够**
4. 注册完成后登录

### 第二步：生成应用密码

> ⚠️ 重要：坚果云的**登录密码不能直接用于第三方应用**，需要单独生成一个「应用密码」。

1. 登录坚果云后，点击右上角头像 → **「账户信息」**
2. 左侧菜单选择 **「安全选项」**
3. 向下滚动到 **「第三方应用密码」** 部分
4. 点 **「添加应用密码」**
5. 名称随便填，比如 `Morandi Schedule`
6. 会生成一串密码（类似 `abc123def`），**复制保存好**（关闭页面后就不能再看了）

### 第三步：在 App 中配置

1. 打开 Morandi Schedule
2. 点底部导航栏 → **「设置」**
3. 找到 **「云同步」** 分类
4. 填写以下三项：

   | 字段 | 填写内容 |
   |:----|:---------|
   | **服务器地址** | `https://dav.jianguoyun.com/dav/` |
   | **用户名** | 你的坚果云**邮箱账号** |
   | **密码** | 上一步生成的**应用密码**（不是登录密码） |
   | **昵称** | 给自己起个名字（例如：`张三`），用于区分多设备 |

5. 点 **「测试连接」** → 提示「连接成功」表示配置正确
6. 打开 **「自动同步」** 开关

### 第四步：验证同步

- 在电脑上添加一条任务或打一次卡
- 手机上打开 Morandi Schedule → 稍等几秒 → 数据自动出现
- 以后**任何一端改了数据，另一端自动同步**

### 多设备注意事项

| 场景 | 说明 |
|:----|:-----|
| **手机 + 电脑** | 两台设备配置相同的 WebDAV 地址、用户名、密码 |
| **多个家庭成员** | 每人注册各自的坚果云，互相独立 |
| **共享习惯打卡** | 需要双方都配置好 WebDAV，然后在「共享空间」中邀请对方 |
| **更换设备** | 新设备配好 WebDAV 后，点一次「手动同步」即可拉取所有数据 |

## 🎯 基本使用

### 添加任务

1. 底部导航 → **「今天」** 或 **「待办」**
2. 点 **「+」** 按钮
3. 填写标题、日期、优先级等
4. 点保存

> 快速添加：在顶部的输入框中直接打字，按回车创建

### 习惯打卡

1. 底部导航 → **「习惯」**
2. 点 **「+ 新建习惯」**
3. 设置习惯名称、频率、目标
4. 每天打卡：点 **「打卡」** 按钮

### 共享习惯（和朋友一起打卡）

1. 底部导航 → **「共享空间」**
2. 创建共享习惯 → 选择要邀请的人
3. 对方在「共享空间」中接受邀请
4. 互相能看到打卡进度和连续天数

### 课表导入

1. 底部导航 → **「设置」**
2. 找到 **「课表管理」**
3. 上传 ICS 格式的课程文件
4. 课程自动显示在周视图中

## ❓ 常见问题

**Q：需要付费吗？**
A：完全免费。只需要注册一个坚果云账号（免费版即可）。

**Q：数据安全吗？**
A：所有数据存在你自己的坚果云里，我们拿不到。还可以开启端到端加密（设置页中配置私有密钥）。

**Q：为什么同步没反应？**
A：检查设置页的 WebDAV 地址和密码是否正确，点「测试连接」。网络问题等 1-2 分钟会自动重试。

**Q：iPhone 怎么用？**
A：Safari 打开网页版 → 分享按钮 → 添加到主屏幕，跟 App 一样用。

**Q：换了手机数据会丢吗？**
A：只要在旧设备上配置过 WebDAV，新设备连接同一个账号，数据自动同步回来，不会丢。

## 🏗️ 自行构建

如果你是开发者，想自己从源码编译：

```bash
git clone https://github.com/yemaozi31-cloud/Morandi-Schedule.git
cd Morandi-Schedule
npm install
npm run dev        # 开发模式
npx tauri build    # 编译桌面端
```

> 需要 Node.js 20+、Rust 1.77+、Android SDK（仅编译 APK 时需要）
