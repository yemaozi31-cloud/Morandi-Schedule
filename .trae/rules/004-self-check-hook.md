# 自检 + 更新日志 强制协议

> 每次修改代码后，必须按此协议执行完整自检 + 更新日志，确认无误后才能结束对话。

## 术语定义

- **日志** = `.trae/rules/001-error-memory.md`（即错误记忆文件/三链记录）
- **三链** = CRUD链路、错误处理链路、数据一致性链路（详见 Step 3）
- **自检** = 构建验证 + 类型诊断 + 功能链路自检
- **更新日志** = 将本次修改记录写入 `001-error-memory.md`

## 核心原则

**每次修改结束前，必须同时完成两件事：自检通过 + 日志更新。缺一不可。**

---

## 执行时机

每次修改涉及以下任一操作后，**必须执行一次完整自检 + 更新日志**：
- 创建新文件
- 修改 `.vue` 文件
- 修改 `.ts` 文件
- 修改 CSS/样式文件
- 修改路由/导航配置

---

## 自检步骤

### Step 1: 构建验证
```
npx vite build 2>&1
```
- [ ] 0 错误
- [ ] 0 警告
- [ ] 模块数与原大致一致（增减合理）

### Step 2: 类型诊断
```
GetDiagnostics（VS Code）
```
- [ ] 0 个诊断错误

### Step 3: 功能链路自检（按修改范围勾选）

#### 3.1 布局/导航修改
- [ ] 桌面端 Sidebar 正常显示，所有图标正确
- [ ] 移动端底部 Tab 正常显示
- [ ] 移动端抽屉菜单正常滑出/关闭
- [ ] 所有路由可正常导航
- [ ] 无 z-index 冲突（Modal / Drawer / Toast 层级合理）
- [ ] 无 emoji 残留（必须用 Icon 组件）

#### 3.2 CRUD 操作修改（三链检查）
- [ ] create 链路：用户操作 → Store → DB → UI 更新 → Toast
- [ ] update 链路：同上
- [ ] delete 链路：确认对话框 → 软删除 → Toast
- [ ] toggle 链路：乐观更新 → DB 写入 → 回滚（失败时）
- [ ] 所有 async 函数有 try/catch
- [ ] IndexedDB 数据直接操作有 Proxy 保护（JSON roundtrip）

#### 3.3 UI/样式修改（必须手机端和桌面端都检查）
- [ ] 桌面端显示正常（>1024px）
- [ ] 平板端显示正常（768-1023px）
- [ ] 手机端显示正常（<768px）
- [ ] 底部安全区域（safe-area-inset-bottom）已处理
- [ ] 删除/关闭按钮不与内容重叠
- [ ] 模态框/弹窗内容不溢出
- [ ] 所有固定尺寸元素（如 `width/height: Npx`）不受 `min-height` 全局规则挤压（如 `responsive.css` 的 `button { min-height: 36px }`）
  - 检查所有 `.color-option`、标签、圆角元素在手机端是否变形
  - 检查所有 `border-radius: 50%` 或 `var(--radius-full)` 的元素宽高是否一致
- [ ] 模态框/弹窗在窄屏下内容不溢出、按钮不重叠
- [ ] flex 布局子项在窄屏下不会被压缩到消失（加 `min-width: 0` / `flex-shrink: 0`）

### Step 4: 更新日志（强制）

将本次修改记录写入 `.trae/rules/001-error-memory.md`，必须包含：
- [ ] **修改内容**：做了什么修改、为什么改
- [ ] **涉及文件**：所有修改过的文件路径
- [ ] **修复方案/实现方式**：关键逻辑描述
- [ ] **预防措施**：后续编码如何避免同类问题

> 如果本次修改是为了修复 Bug，还需在日志中记录的"已知漏洞清单"中更新状态。

**在结束本次对话前，必须确认：**
1. ✅ Step 1 构建验证通过（0 错误）
